# Lukas Hermann | Strategic B2B Framework

### Core Content Architecture

- Identifies SaaS opportunities through workflow observation rather than idea generation. The process begins by studying how people in adjacent departments spend their time, identifying repetitive manual tasks, and evaluating whether those workflows can be automated with software.
- Uses direct user proximity as the primary validation mechanism. Product ideas are prioritized only when a specific person can immediately be identified as a likely user. If no concrete user comes to mind during ideation, the opportunity is treated as weak regardless of market size.
- Advocates for extremely narrow MVP construction focused on a single core user outcome. Rather than building multiple features, MVPs should replicate only the essential experience required to solve the target problem and should be treated as disposable prototypes rather than production software.
- Builds distribution around existing user workflows instead of category-level keywords. Content, landing pages, and SEO assets are designed around the tools, processes, and search behaviors users already employ rather than generic industry terminology.

### Operational Metrics & Tactics

- Uses a staged validation process:
  - Identify a repetitive workflow.
  - Build the smallest possible solution.
  - Release it free to likely users.
  - Measure whether users return.
  - Rebuild properly only after repeat usage is demonstrated.
- Treats repeat usage as the primary early-stage product-market-fit signal. The key metric is not acquisition but whether users return voluntarily after their initial interaction with the product.
- Acquires early users through highly niche Reddit communities rather than broad startup or entrepreneur audiences. Initial posts are framed as honest requests for feedback on free tools rather than promotional launches, increasing engagement and reducing resistance.
- Uses customer interviews to uncover non-obvious SEO opportunities. Instead of relying solely on keyword tools like Ahrefs or SEMrush, users are asked what tools they previously used, how they solved the problem before, and what workflows they followed. These answers become the basis for high-converting content and search pages.

### Distinctive Insight

- The strongest SaaS opportunities are hidden inside existing workflows. Rather than searching for startup ideas, founders should observe how people currently work, identify repetitive manual processes, and validate solutions through repeat user behavior before investing in full product development.
